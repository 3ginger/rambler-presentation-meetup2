Keep it simple, stupid (KISS)

Бритва Оккамы: «Не следует множить сущее без необходимости»

Есть только один путь развития - итеративный

Ничто не береться из воздуха

Все имеет свое начало (а иногда и не одно)

Вбирая разные пропорции своих предков образуется множество

Через какое то время можно подумать, что все пораждается из воздуха

Рефакторинг как яркий пример итеративного подхода

  • Говнокод - не всегда плохо
  • Прототип - это корни рефакторинга
  • Опыт побеждает технологии *
  • Рефакторинг побеждает опыт

Что же дает рефакторинг?

  • Рефакторинг дает право экспериментировать
  • Рефакторинг дает понимание, что нужно структурировать, а что не стоит
  • Рефакторинг дает общую картину о проекте
  • Рефакторинг дает способность стирать свой код
  • Рефакторинг дает многогранность мышления
  • Рефакторинг дает право ощутить проект на любом этапе

Что же забирает рефакторинг?

  • Рефакторинг забирает право писать совершенный код (совершенный значит бесконечный)
  • Рефакторинг стирает рамки проектов

Формула для выбора между новой технологией и хорошо знакомой старой технологией *

  • time_dev - время на разработку проекта
  • coeff_exp - коэффициент опыта ( меньше лучше )
  • time_tech - время на изучение новой технологии
  • coeff_tech - коэффициент технологии ( меньше лучше )

time_dev * coeff_tech + time_tech < time_dev * coeff_exp

Технологии и индивидуальный выбор

  • Задачи выбириют технологии
  • Задачи пораждают технологии
  • Технологии не могут существовать без контекста
  • Не бывает универсального удобства
  • Индивидуальность человека обусловлена воспитанием
  • Поиск компромисса
